# Mission

The mission of PlayOS is to make developing console-quality games **simple,
portable, and open**.

## The three promises

**Developers should write games, not operating systems.**
A developer should be able to write an application against the PlayOS
Platform API, package it once, and target many devices through
well-defined backends and capabilities — without learning the entire
Linux input, audio, and display stack.

**Players should get a fast, controller-first console experience.**
PlayOS should feel like a console: instant, controller-first, touch-aware,
with no visible desktop, terminal, or window manager.

**Hardware manufacturers should adopt PlayOS without reinventing the
stack.**
A vendor should be able to bring PlayOS to new hardware by writing a
device profile and, where needed, a backend — not by forking the platform.

## What success looks like

- A game built once runs on handheld PCs, Linux PCs, ARM boards, and
  supported SDK targets.
- The Platform API stays stable long enough that developers trust it.
- Communities, schools, and OEMs can self-host the services they depend
  on.
- New devices are added by writing a profile, not by changing the SDK.

## The motto

> **Write Once. Play Anywhere. Own Everything.**

- **Write Once** — one Platform API, many targets.
- **Play Anywhere** — runtime devices and SDK targets from the same code.
- **Own Everything** — open specification, open reference code, and
  self-hostable services.
